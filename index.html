Method 0: Blocking TikTok Domains Directly in Router Settings

Many home routers allow you to block specific domains or websites directly in their settings, without any extra software. This is the simplest way if your router supports it.

  • Log in to your router's admin panel (usually by entering 192.168.0.1 or 192.168.1.1 in your browser).
  • Look for a section called Access Control, Parental Controls, Website Filtering, or Block Sites.
  • Find the option to block domains or websites.
  • Copy the domains from the "Main Domains" or "All Domains" list above and paste them into the block list (one per line, if required).
  • Save/apply the settings and reboot the router if necessary.
  • Note: Not all routers support domain blocking, and some may have a limit on the number of domains you can enter. If your router does not have this feature, use one of the other methods below.

Method 1: Router Level Blocking (Recommended for Whole Network)

Many modern routers support custom DNS servers or DNS filtering features. Services like Pi-hole, AdGuard Home, or NextDNS are popular choices.

  • Pi-hole / AdGuard Home:
    1. Set up Pi-hole or AdGuard Home on a device on your network (like a Raspberry Pi).
    2. Access the admin interface.
    3. Navigate to the blocklist or domain management section.
    4. Copy either the "Main Domains" or "All Domains" list from this page.
    5. Paste the domains into the blocklist section. Ensure each domain is on a new line.
    6. Save the changes and update gravity/lists if required.
    7. Configure your router's DHCP settings to use the Pi-hole/AdGuard Home device as the DNS server for your network.
  • NextDNS / Cloudflare Gateway / Similar Cloud Services:
    1. Sign up for a service like NextDNS or Cloudflare Zero Trust Gateway.
    2. In their dashboard, navigate to the "Denylist" or "Block" section.
    3. Copy and paste the desired domain list from this page.
    4. Configure your router or individual devices to use the DNS server addresses provided by the service.
  • Router's Built-in Filtering: Some routers have basic domain blocking features. Check your router's manual. Copy and paste the domains if supported, though limitations may apply.

Method 2: Device Level Blocking

You can configure DNS settings directly on individual devices.

  • Windows:
    1. Edit the 'hosts' file (C:\Windows\System32\drivers\etc\hosts) with administrator privileges.
    2. For each domain you want to block, add a line like: 0.0.0.0 domain.com or 127.0.0.1 domain.com.
    3. Copy the domains from the list and format them accordingly.
    4. Save the file. A restart or running ipconfig /flushdns in Command Prompt might be needed.
    5. Alternatively, use DNS filtering software like AdGuard for Windows or configure a secure DNS provider (like NextDNS) in Windows settings if available.
  • macOS:
    1. Edit the 'hosts' file (/etc/hosts) using sudo nano /etc/hosts in Terminal.
    2. Add lines like 0.0.0.0 domain.com for each domain.
    3. Save the file (Ctrl+O, Enter, Ctrl+X). Flush DNS cache using sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder.
    4. Alternatively, use DNS filtering software like AdGuard for Mac or configure a secure DNS profile (e.g., for NextDNS).
  • iOS / iPadOS:
    1. Direct hosts file editing is not possible without jailbreaking.
    2. Use DNS filtering apps like AdGuard for iOS or NextDNS from the App Store. Configure the blocklists within the app.
    3. Alternatively, configure a custom encrypted DNS profile (DoH/DoT) in Settings > Wi-Fi > (Your Network Info) > Configure DNS, or via a configuration profile for services like NextDNS.
  • Android:
    1. Direct hosts file editing usually requires root access.
    2. Use DNS filtering apps like AdGuard for Android, NextDNS, or RethinkDNS from the Play Store or F-Droid. Configure the blocklists within the app.
    3. Alternatively, configure "Private DNS" in Network Settings (Android 9+). Enter the hostname for a DNS-over-TLS provider (like one from NextDNS) that uses your configured blocklist.
